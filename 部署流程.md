# TeamHub 生产环境部署流程

## 你需要准备的东西

| 项目 | 说明 | 费用 |
|------|------|------|
| 云服务器 1台 | 腾讯云/阿里云 轻量应用服务器 2核4G | ~68元/月 |
| 域名 1个 | `.com` / `.cn`，在域名商处购买 | ~50元/年 |
| SSL证书 | Let's Encrypt 免费 | 0元 |

---

## 第一步：买服务器

1. 打开 [腾讯云轻量服务器](https://cloud.tencent.com/product/lighthouse) 或 [阿里云轻量](https://www.aliyun.com/product/swas)
2. 选择配置：
   - **CPU**: 2核
   - **内存**: 4GB
   - **系统盘**: 60GB SSD
   - **带宽**: 4-6Mbps
   - **系统**: Ubuntu 22.04 LTS
3. 付款后等待1-2分钟，服务器就绪

> **为什么选 Ubuntu 22.04？** Docker 对其支持最好，社区资料最多。

---

## 第二步：登录服务器

打开你的终端（Windows 用 PowerShell 或 CMD），用服务器 IP 登录：

```bash
ssh root@你的服务器IP
```

首次登录会提示确认，输入 `yes`。

---

## 第三步：安装 Docker

一行命令搞定：

```bash
curl -fsSL https://get.docker.com | sh
```

等待完成，然后：

```bash
# 让 docker 命令不需要 sudo
usermod -aG docker $USER
exit
```

再 `ssh root@你的IP` 重新登录一次，Docker 就装好了。

---

## 第四步：上传项目代码

**在你的电脑上**（不是服务器），把项目上传到服务器：

```bash
# 打包项目（不包含 node_modules 和 venv）
cd C:\Users\le\projects\teamhub
tar -czf teamhub.tar.gz \
    --exclude=node_modules \
    --exclude=venv \
    --exclude=__pycache__ \
    --exclude=teamhub.db \
    --exclude=*.log \
    .

# 上传到服务器
scp teamhub.tar.gz root@你的服务器IP:/root/
```

**在服务器上**解压：

```bash
mkdir -p /root/teamhub
tar -xzf /root/teamhub.tar.gz -C /root/teamhub
cd /root/teamhub
```

---

## 第五步：配置环境变量

```bash
cd /root/teamhub

# 复制环境变量模板
cp backend/.env.example backend/.env

# 生成随机 JWT 密钥
openssl rand -base64 48

# 编辑配置
nano backend/.env
```

把 `.env` 改成这样：

```env
JWT_SECRET_KEY=刚才openssl生成的那一串随机字符
DATABASE_URL=sqlite:///./teamhub.db
ALLOW_SEED=false
```

保存：`Ctrl+O` → 回车 → `Ctrl+X`

---

## 第六步：部署

```bash
cd /root/teamhub
chmod +x deploy.sh
./deploy.sh
```

这个脚本会自动：
1. 构建前端（React → 静态文件）
2. 拉取 Docker 镜像
3. 启动 Nginx + 后端 + 前端

完成后访问 `http://你的服务器IP` 就能看到 TeamHub。

---

## 第七步：绑定域名 + SSL（HTTPS）

### 7.1 域名解析

在域名管理后台添加一条 **A 记录**：

| 类型 | 主机记录 | 记录值 |
|------|---------|--------|
| A | @ | 你的服务器IP |
| A | www | 你的服务器IP |

### 7.2 配置 HTTPS

用 Caddy 替代 Nginx 最简单（自动申请 SSL 证书）。或者继续用 Nginx + Certbot：

```bash
# 安装 certbot
apt install -y certbot python3-certbot-nginx

# 先把域名配进 nginx
nano nginx/nginx.conf  # 把 server_name localhost 改成你的域名

# 重启
docker-compose restart nginx

# 申请证书
certbot --nginx -d 你的域名.com -d www.你的域名.com
```

证书 90 天自动续期，不用管。

---

## 第八步：首次登录

1. 访问 `https://你的域名.com/login`
2. 需要先创建管理员账号。临时开启 seed 功能：

```bash
cd /root/teamhub
# 编辑 .env，把 ALLOW_SEED 改为 true
nano backend/.env
# 改成 ALLOW_SEED=true

# 重启后端
docker-compose restart backend

# 用 curl 调用 seed（或浏览器访问一次即可）
curl -X POST https://你的域名.com/api/seed

# 重要：改回来！
# 把 .env 的 ALLOW_SEED 改回 false
nano backend/.env
docker-compose restart backend
```

3. 用默认账号登录：
   - 管理员：用户名 `管理员` 密码 `admin123`
   - 测试用户：用户名 `测试成员` 密码 `user123`

4. **登录后立刻改密码！**

## 日常运维

### 常用命令

```bash
cd /root/teamhub

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f backend   # 后端日志
docker-compose logs -f nginx     # 访问日志

# 重启
docker-compose restart backend   # 只重启后端
docker-compose down && docker-compose up -d  # 全部重启

# 更新代码
git pull                         # 拉最新代码
./deploy.sh                      # 重新构建部署

# 备份数据库
cp backend/teamhub.db backups/teamhub_$(date +%Y%m%d).db

# 磁盘空间
df -h
```

### 设置自动备份

```bash
crontab -e
```

加一行（每天凌晨3点备份）：

```
0 3 * * * cp /root/teamhub/backend/teamhub.db /root/teamhub/backups/teamhub_$(date +\%Y\%m\%d).db
```

## 常见问题

**Q: 100人够用吗？**
A: 够。TeamHub 是内部协作平台，100人不会同时操作。2核4G 跑 Nginx + FastAPI + SQLite 绰绰有余。

**Q: SQLite 扛得住吗？**
A: 100人以内的内部系统完全够。如果以后需要换 PostgreSQL，项目已经支持了——把 `DATABASE_URL` 改成 PostgreSQL 连接串就行。

**Q: 服务器需要备案吗？**
A: 如果用国内服务器（阿里云/腾讯云），域名需要备案（约20天）。不想备案可以买香港节点的服务器，延迟高一点但可以直接用。
